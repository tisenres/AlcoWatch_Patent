## Project Summary
A Python/Streamlit middleware system that sits between raw business event streams and LLM-based multi-agent pipelines. It builds a knowledge graph from supply chain events, computes semantic deltas between snapshots, scores changes by business saliency, and forwards only high-importance changes to LLM agents — achieving 3x–87x token compression with 94% detection accuracy maintained.

## Report Structure

### Chapter 1 — Introduction
**Sections:**
- 1.1 Background — LLM adoption in enterprise; token cost crisis (GPT-4o = $0.03/1K tokens; 1000 queries/day = $450–$1500/month baseline)
- 1.2 Motivation — Three trends: (a) LLM cost scales with context size, (b) knowledge graphs mature for business data, (c) saliency-based filtering proven in information retrieval
- 1.3 Project Scope — Software only; synthetic supply chain domain; no new hardware; Python/Streamlit/NetworkX stack
- 1.4 Problem Statement — Four gaps: no symbolic preprocessing layer for LLMs, no saliency-aware context compression, no supply-chain-specific multi-agent middleware, no open benchmarked system
- 1.5 Objectives — Five concrete: (1) build neuro-symbolic pipeline, (2) achieve 10x+ compression with ≥90% recall, (3) implement Streamlit dashboard, (4) benchmark against naive baseline, (5) demonstrate multi-agent reasoning on distilled briefings
- 1.6 Connection to AlcoWatch — Same author's prior work; demonstrates wearable → vehicle actuation; this project generalizes the middleware concept from physiological signals to business signals

**Key numbers to cite:** 87x max compression, 94% recall, <20ms overhead, 52–99% cost savings

### Chapter 2 — Literature Review
**Sections:**
- 2.1 LLM Token Economics — Context window limits, cost scaling, RAG as partial solution, limitations of naive chunking
- 2.2 Knowledge Graphs for Business Intelligence — NetworkX, supply chain graph modeling, centrality measures (degree, betweenness), graph-based anomaly detection
- 2.3 Saliency Scoring and Information Filtering — Attention mechanisms, information-theoretic filtering, threshold-based selection; cite relevant NLP and information retrieval work
- 2.4 Multi-Agent LLM Systems — Agent frameworks, tool use, role-based agents; cost as the key deployment barrier
- 2.5 Gap Analysis — No existing work combines: symbolic delta computation + neural saliency scoring + structured distillation into LLM-ready briefings in a single open pipeline

### Chapter 4 — Methodology
**Sections:**
- 4.1 Synthetic Supply Chain Data Model — 10 event types (sales, restocks, shipments, returns, price changes, supplier delays, quality issues, demand spikes, stockouts, new contracts); fixed topology (products × suppliers × warehouses × regions)
- 4.2 Knowledge Graph Construction — Node types: product, supplier, region, warehouse; edge types: supplies, sold_in, stored_at, ships_to; anomaly_score and reliability attributes; 30 nodes, 368 edges at steady state
- 4.3 Semantic Delta Computation — Structural diff between T and T+1 graph snapshots; categories: new nodes, removed nodes, changed attributes, new edges, removed edges
- 4.4 Saliency Scoring — Formula: saliency = α×(degree_centrality) + β×(betweenness_centrality) + γ×(attribute_magnitude); threshold θ=0.4; empirically tuned on ground truth patterns
- 4.5 Distillation Format — Business Intelligence Briefing structure: CRITICAL/IMPORTANT/NOISE classification; relationship context included; target 600–900 tokens
- 4.6 Evaluation Methodology — Ground truth patterns (4 known anomalies injected), ablation (naive vs aggregation vs full pipeline), quality parity test (recall comparison)

**Table:** FSM equivalent — distillation severity levels (CRITICAL/IMPORTANT/INFORMATIONAL/NOISE) with thresholds and agent routing
